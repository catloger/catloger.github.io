Git 和 Github

2020

什么是Git
  Git是一个免费、开源的版本控制软件

什么是版本控制系统
  版本控制是一种记录一个或若干个文件内容变化，以便将来查阅特定版本修订情况的系统。
其中，系统具体功能：

  • 记录文件的所有历史变化；
  • 随时可恢复到任何一个历史状态；
  • 多人协作开发或修改；
  • 错误恢复；

什么是Github
  Github是全球最大的社交编程及代码托管网站（https://github.com/），可以在这里学到优秀的开源项目。
  Github可以托管各种git库，并提供一个web界面（用户名.github.io/仓库名）,本站便是通过这种机制进行创建的。

Github和Git是什么关系
  Git是版本控制软件，Github是项目代码托管的平台，借助git来管理项目代码。

Git初始化及仓库创建和操作
基本信息设置：

  • 设置用户名
    git config --global user.name 'zhangsan'
  • 设置用户名邮箱
    git config --global user.email '123456@qq.com'

在文件内初始化git（创建git仓库）
git init

向仓库添加文件
touch a1.php # 创建a1.php文件到工作目录
git add a1.php # 添加a1.php到暂存区
git commit -m '第一次提交文件' # 添加a1.php到仓库

修改仓库文件
vi a1.php
git status
git add a1.php
git status

删除仓库文件
rm -rf a1.php
git rm a1.php
git commit -m '通过git删除仓库文件'

将远程仓库（github对应的项目）复制到本地
git clone 仓库地址

将本地仓库同步到git远程仓库中
touch b.php
git add b.php #添加到暂存区
git commit -m '提交到仓库'
git push

排错：The requested URL returned error: 403 Forbidden while accessing
解决如下：
私有项目，没有权限，输入用户名密码，或者远程地址采用这种类型：
vi .git/config
将[remote "origin"]
url = https://github.com/用户名/仓库名.git
修改为：
[remote "origin"]
url = https://用户名:密码@github.com/用户名/仓库名.git
